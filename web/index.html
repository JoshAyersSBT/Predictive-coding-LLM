    Backpropagation vs Predictive Coding

    Two ways to assign credit through a layered language model.

    Standard Backpropagation

    Input x
    forward
    Layers f1...fL
    loss
    Token CE
    global gradient flows backward
    h_l = f_l(h_{l-1}; theta_l) L_BP = CE(y, softmax(W h_L)) theta_l <- theta_l - eta * dL_BP / dtheta_l

    Backprop waits for the final token loss, then differentiates that scalar objective through every layer by the chain rule.

    Predictive Coding Schema

    Layer h_l
    predicts
    Next h_{l+1}
    error
    e_l = h_{l+1} - p_l(h_l)
    local prediction errors shape representations
    p_l = g_l(h_l; phi_l) L_PC = sum_l mean(||stopgrad(h_{l+1}) - p_l||^2) L_total = L_BP + lambda * L_PC

    Our current model still uses gradient descent, but adds local hidden-state prediction errors so each layer is pressured to make the next layer easier to predict.

    Practical difference in this tool

    Backprop optimizes the final next-token objective. Predictive coding adds an auxiliary layer-to-layer consistency signal, tracked as PC loss, while LM loss tracks language modeling quality.
